## ТЗ: Поиск по тексту карточек (каталог)

### Цель

Добавить текстовый поиск по карточкам в режиме **«Каталог»**, чтобы пользователь мог быстро находить вопросы по ключевым словам.

---

### Область применения

* Только режим **«Каталог»**
* Поиск применяется **вместе** с фильтрами:

  * тема
  * уровень сложности

---

### Функциональные требования

#### Поведение поиска

* Текстовое поле ввода (input)
* Поиск выполняется по:

  * **вопросу**
  * **ответу**
* Совпадение:

  * подстрока
  * без учёта регистра
* Пустая строка → поиск отключён

#### Логика

* Поиск применяется **к уже отфильтрованным карточкам**
* Порядок карточек не меняется
* Результаты обновляются **в реальном времени** при вводе

---

### UI/UX требования

* Одно строковое поле «Поиск»
* Без кнопки «Найти»
* Без автокомплита
* Без подсветки совпадений (на первом этапе)
* При отсутствии результатов:

  * показать текст «Ничего не найдено»

---

### Ограничения

* ❌ Не использовать серверный поиск
* ❌ Не использовать сторонние библиотеки
* ❌ Не использовать полнотекстовый поиск
* ❌ Не учитывать морфологию и синонимы

---

### Производительность

* Поиск выполняется на клиенте
* Объём данных: ~400–500 карточек
* Задержки и лаги недопустимы
* Допустимо использовать debounce (опционально)

---

### Accessibility

* Поиск должен быть доступен с клавиатуры
* Фокус в поле ввода не сбрасывается при обновлении списка

---

### Критерии приёмки

* Ввод текста уменьшает список карточек
* Поиск работает одновременно с фильтрами
* Очистка поля возвращает полный список
* Интерфейс не перезагружается
