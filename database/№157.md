

Когда использовать `pandas.merge`, а когда `pandas.join`?

---

`merge` — универсальный способ объединения DataFrame:
- объединяет по столбцам, даже если их имена различаются
- подходит для большинства SQL-подобных join’ов (`inner`, `left`, `right`, `outer`)
- явно указывает ключи (`left_on`, `right_on`)
    
`join` — упрощённый синтаксис:
- по умолчанию объединяет по индексам
- удобен, когда ключ уже является индексом
- короче и читабельнее в простых случаях
