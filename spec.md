## Задумка

Веб-тренажёр для подготовки к собеседованиям аналитика (DA / DS / ML), построенный вокруг одного источника истины — **Markdown-карточек вопрос/ответ**.

### Базовая философия проекта

* **Контент важнее UI**
  Все знания живут в `.md`, тренажёр — лишь оболочка.
* **Минимум магии**
  Никаких БД, серверов, фреймворков.
  Всё читаемо, воспроизводимо, контролируемо.
* **Статика как база, ИИ как опция**
  90% функционала работает без интернета и API.
  GPT подключается *только* в режиме интервью.

---

### Структура данных

**В корне проекта:**

* папка `database/`
  * `*.md` — карточки (1 файл = 1 вопрос)
  * `Train to DA.md` — *карта тем*, логическая группировка
* `build.py`
  * парсит `.md`
  * собирает единый `data.json`
* `data.json`
  * единственный источник данных для фронта
  * из него строится всё: каталог, карточки, блиц, интервью

Markdown не меняется, не загрязняется UI-логикой, не влияеи и независит от веба.


### Фильтры

* **Тема карточек**
  Берётся из структуры `Train to DA.md`
* **Уровень сложности**
  (`#lvl_1`, `#lvl_2`, `#lvl_3`)
  субъективный, выставлен по ощущениям.


### Режимы тренажёра (4 независимых сценария)

#### 1. **Каталог** (главная страница)

* статичный порядок карточек
* два фильтра (тема / уровень)
* краткая статистика:

  * сколько карточек выбрано
  * распределение по уровням
* каждая карточка:

  * тема, уровень, номер
  * вопрос
  * **заблюренный ответ**
  * кнопка «Показать ответ»

Назначение: *просмотр базы, повторение, чтение, навигация по знаниям*


#### 2. **Карточки**

* одна карточка за раз
* выбор темы
* **рандомизация порядка**
* кнопки:

  * «Показать ответ»
  * «Следующий вопрос»

Назначение: *активное вспоминание, фокусированный прогон по теме*


#### 3. **Блиц**

* тема и уровень **не выбираются**
* вопросы приходят **рандомно со всей базы**
* пользователь:

  * формулирует ответ в голове / вслух
  * открывает ответ
  * сам оценивает себя кнопками:

    * «Знал»
    * «Нужно повторить»
* собирается **локальная статистика сессии**:

  * количество вопросов
  * точность
  * серия правильных ответов

Назначение: *быстрая самопроверка, «разогрев», без давления и ИИ*


#### 4. **Интервью (LLM-режим)**

Единственный режим с внешним API.

* интерфейс похож на блиц
* **1 вопрос → 1 большой ответ**
* ответ можно:

  * напечатать
  * наговорить (несколько войсов → склеиваются)
* эталонный ответ **скрыт**
* всё отправляется в LLM:

  * вопрос
  * эталон
  * ответ пользователя
* LLM возвращает:

  * оценку `0–10`
  * короткий фидбек
* статистика сессии обновляется на основе **оценок ИИ**

Назначение: *симуляция собеседования на минималках, без диалога и уточняющих вопросов*

### Техническая архитектура

**Фронт**
* чистый HTML / CSS / JS
* `fetch()` → `data.json`
* GitHub Pages

**ИИ-часть**
* Serverless Function (Python)
  * Vercel / Netlify
  * секреты в env vars
* фронт → HTTPS → function → GPT → JSON-ответ

---

### UI / дизайн

* минимализм
* ч/б / тёмные оттенки
* без визуального шума
* позже возможен стиль **ретро 80-х (synthwave)** — как опциональный скин, не влияющий на логику
