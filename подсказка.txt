
1) создать пустую папку с новым проектом

2) скорипровать/вставить данный шаблон, либо клонировать репо с гитхаба - git clone <URL>

3) проверить наличие:

    .cursor           - правила и роли для курсора в рамках проекта
    .vscode           - настройки как проводить импорты, настройки шрифтов, хлебные крошки и тд.
    .pylintrc         - настроека для пайлинтера (подсказки, pip8, список не нужных напоминалок/ошибок)
    prompt_config.sh  - настройка для вывода красивого промта в терминале
    .gitignore        - список того что игноритьь в системе контроля версий git
    LICENSE           - стандартная лицензия (хз зачем, пока просто так)


4) venv

создать папку venv с виртуальным окружением через терминал

    python -m venv venv
    python3.11 -m venv venv

запустить виртуальное окружение

    source venv/Scripts/activate  - для GitBash
    venv\Scripts\activate         - для Windows

    deactivate                    - выйти из вируального окружения

проверить

    ls                            - фактическое расположение
    which python                  - откуда берётся Python
    python -m pip -V              - откуда берётся pip


6) version

проверить актуальные версии

    python --version
    pip --version
    git --version
    docker --version


7) requirements

    pip freeze > requirements.txt         - создать файл requirements.txt из ВСЕХ зависимостей в env

    pip install -r requirements.txt       - скачать все расширения в env из файла requirements.txt

    pip install pipreqs                   - скачать pipreqs
    pipreqs . --force                     - создать файл requirements.txt из ИСПОЛЬЗУЕМЫХ зависимостей проекта

    pip uninstall <github>                - удалить определенную зависимость из env


8) Git

    git init
    git pull

    git add .
    git commit -m "text_commit"

    git push

    rm -rf .git                            - отключить систему контроля версий Git


9) .exe

создание исполняемого файла

    pip install pyinstaller            - качаем необходимую библиотеку
    version_info.txt                   - создаем текстовый файл с данными об авторстве

    pyinstaller --onefile --windowed --add-data "images/*;images" --version-file version_info.txt main.py


10) Jupyter

в IDE работать с файлами .ipynb не удобно.
через встроенную поддержку Cursor виджиты не работают, все равно нужен Jupyter.

    where jupyter                           - проверить наличие и расположение библиотеки

    cd <field>                              - перейти в папку с нужным файлом

    jupyter notebook poker_simulation.ipynb - запуск Jupyter Notebook (откроется браузере)
    jupyter lab poker_simulation.ipynb      - запуск JupyterLab (более современный)
